Low Level Design (LLD)

Focus Area

It majorly focuses on classes and objects within a system.

Goals of LLD

  • To write clean code
  • Code should be flexible and maintainable
  • Code should be easy to test

Creational Design Patterns

It controls the object creation.

  • Singleton
  • Builder
  • Factory
  • Abstract Factory
  • Object Pool
  • Prototype

Structural Design Patterns

Focuses on how different classes / objects are arranged together so that a larger problem can be solved in a flexible way.

  • Decorator
  • Proxy
  • Composite
  • Adapter
  • Bridge
  • Façade
  • Flyweight

Behavioral Design Patterns

Focuses on how different objects communicate and interact with each other.

  • State
  • Strategy
  • Observer
  • Chain of Responsibility
  • Template
  • Iterator
  • Interpreter
  • Command
  • Visitor
  • Mediator
  • Memento
  • Null Object

has-a & is-a Relationship

is-a Relationship

This represents inheritance.

has-a Relationship

Shows a link between two objects.

  • House has rooms
  • Library has books
  • School has students

Types of Association

Weak Relationship (Aggregation)

Existence of one object is not dependent on another. 


public class Library {
    List<Books> books;
}

Strong Relationship (Composition)

Existence of one object is dependent on another. 


public class House {
    List<Rooms> rooms;

    public House() {
        rooms = new ArrayList<>();
        rooms.add(new Room("Living Room"));
        rooms.add(new Room("Bedroom"));
    }
}
